Building Custom Tab Bars with SwiftUI & Matched Geometry Effect

June 12, 2026 · 8 min read · Xcode 15+

SwiftUI's built-in TabView is convenient, but it doesn't give you much control over the tab bar's appearance. In this tutorial we'll build a fully custom animated tab bar using matchedGeometryEffect — the same technique behind the fluid indicator animations you see in apps like Fantastical and Craft.

Prerequisites: Xcode 15+, iOS 17+. Familiarity with basic SwiftUI views and @State is assumed.

How matchedGeometryEffect works

matchedGeometryEffect links two or more views with the same id within a shared Namespace. When one view appears or its geometry changes, SwiftUI automatically interpolates the size and position between the matched views — giving you smooth, physics-based motion for free.

The key insight: we're not moving a view. We're letting SwiftUI morph a background shape from one tab to another. The shape stays in the view hierarchy; only its position changes.

Step 1 — Define the tab model

Swift 
enum Tab: String, CaseIterable {
    case home    = "house.fill"
    case explore = "safari.fill"
    case library = "books.vertical.fill"
    case profile = "person.fill"

    var label: String {
        switch self {
        case .home:    return "Home"
        case .explore: return "Explore"
        case .library: return "Library"
        case .profile: return "Profile"
        }
    }
}

Step 2 — Build the tab bar view

The trick is a Namespace and a single RoundedRectangle acting as the indicator. We stamp it with .matchedGeometryEffect(id: "indicator", in: namespace) on whichever tab is selected. When selection changes, SwiftUI slides the shape across automatically.

Swift 
struct CustomTabBar: View {
    @Binding var selection: Tab
    @Namespace private var namespace

    var body: some View {
        HStack(spacing: 0) {
            ForEach(Tab.allCases, id: \.self) { tab in
                TabItem(
                    tab: tab,
                    isSelected: selection == tab,
                    namespace: namespace
                )
                .onTapGesture {
                    withAnimation(.spring(response: 0.35, dampingFraction: 0.7)) {
                        selection = tab
                    }
                }
            }
        }
        .padding(.horizontal, 8)
        .padding(.vertical, 10)
        .background(
            RoundedRectangle(cornerRadius: 28)
                .fill(.ultraThinMaterial)
                .shadow(color: .black.opacity(0.12), radius: 20)
        )
        .padding(.horizontal, 20)
    }
}

Step 3 — The tab item

Swift 
struct TabItem: View {
    let tab: Tab
    let isSelected: Bool
    var namespace: Namespace.ID

    var body: some View {
        VStack(spacing: 4) {
            ZStack {
                if isSelected {
                    Capsule()
                        .fill(Color.accentColor.opacity(0.15))
                        .frame(width: 52, height: 32)
                        .matchedGeometryEffect(id: "indicator", in: namespace)
                }
                Image(systemName: tab.rawValue)
                    .font(.system(size: 18, weight: .medium))
                    .foregroundStyle(isSelected ? .accentColor : .secondary)
                    .scaleEffect(isSelected ? 1.1 : 1.0)
            }
            Text(tab.label)
                .font(.system(size: 10, weight: .medium))
                .foregroundStyle(isSelected ? .accentColor : .secondary)
        }
        .frame(maxWidth: .infinity)
    }
}

Step 4 — Wire it up

Swift 
struct ContentView: View {
    @State private var selectedTab: Tab = .home

    var body: some View {
        ZStack(alignment: .bottom) {
            TabContent(selection: selectedTab)
                .ignoresSafeArea()

            CustomTabBar(selection: $selectedTab)
                .padding(.bottom, 12)
        }
    }
}
Tip: Use .spring(response:dampingFraction:) rather than .easeInOut for the animation. A lower dampingFraction (0.6–0.7) gives the characteristic iOS bounce that feels native.

Going further

  • Add a badge overlay using .overlay with a Circle
  • Animate the icon with .symbolEffect(.bounce) on selection (iOS 17+)
  • Swap the Capsule indicator for a top-edge underline using a pinned Rectangle
  • Support Dynamic Type by using relative font sizes and flexible widths

Try tweaking the spring parameters — a higher dampingFraction (close to 1.0) gives a crisp snap, while lower values feel more playful. There's no single right answer; match the character of your app.
